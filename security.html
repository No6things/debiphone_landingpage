Security Page

  • The Debiphone Application works offline, meaning it does not need to connect to any Bank for authorization. In doing this it avoids potential hacker attacks and online transgressions
  • The User’s personal data are not stored in the User’s Mobile device. They are held in Debiphone Servers hosted in a highly secured Data Center.
  • No User can access the Application without an Individual Access code which acts in accordance with the User’s Mobile number.
  • Users can setup several Mobile numbers where they can use the Application, but only one active at any time.
  • The Application requires to set a GREEN code for regular transactions and a RED code for “Alert” impending transactions to be used in the case of an assault. This RED code limits available balance to a random 2 digit lower amount setup by the User him/herself
  • The RED code will also be to set the GPS signal on your device and alert authorities of an ongoing dangerous situation
  • Balance will only show on the device after the Authorization Code has been entered
  • Bank compensation is done several times a day at randomly selected times
  • Compensation at the Front-End and the Back-End checks transactions sequence and account balances to validate last movements and detect possible alterations. There is no Clearance checks at the front-end level.
  • The Virtual Holding Account is identified by a combination of your Phone Number, the Username, and the Device’s ID. Hence it can only be accessed from a single particular device
